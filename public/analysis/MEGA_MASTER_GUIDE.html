Introduction to ICT

Foundation concepts of Information & Communication Technology

📖 Key Concepts

🔹 What is ICT?

Information and Communication Technology (ICT) refers to all technologies used to handle telecommunications, broadcast media, intelligent building management systems, audiovisual processing, and network-based control systems.

🔹 Data vs Information

Data: Raw, unprocessed facts (e.g., 75, "Saman", 2026)

Information: Processed, meaningful data (e.g., "Saman scored 75 marks in 2026")

Information = Data + Processing + Meaning

📊 ICT Application Areas

AreaExamples
EducationE-learning, LMS, online classes, smart boards
HealthcareTelemedicine, electronic health records, MRI
BankingATMs, online banking, mobile payments
BusinessE-commerce, inventory systems, CRM
GovernmentE-government services (G2C, G2B, G2E, G2G)
AgricultureSmart farming, weather prediction

🏛️ E-Government Services

Types of E-Government:

  • G2C (Government to Citizen): Online ID applications, exam results, revenue licenses
  • G2B (Government to Business): Tax filing, business registration
  • G2E (Government to Employee): Salary slips, leave management
  • G2G (Government to Government): Inter-department data sharing

📝 Past Paper Practice

Q1 (MCQ - 2019)

Which of the following are related to G2C (Government to Citizen) services in Sri Lanka?

A - viewing G.C.E. (O/L) results online
B - ordering food items online
C - renewing vehicle revenue licenses online

✅ Answer: A and C only

Explanation: Both viewing exam results and renewing licenses are services provided by the government to citizens. Ordering food is a private commercial service (e-commerce), not G2C.

Q2 (MCQ - 2020)

The three main functions of an information system are:

✅ Answer: Input, Process, Output

Memory Tip: Remember "IPO" — Input → Process → Output. This applies to ALL information systems.

🎯 Marking Scheme Tips

  • For "examples" questions → Give specific real-world examples
  • For "G2C/G2B" → Always mention "Government" in your answer
  • For "Information vs Data" → Use definitions PLUS examples

Fundamentals of Computer System

Hardware, components, and how computers work

📖 Key Concepts

🔹 Computer System = Hardware + Software + Liveware
  • Hardware: Physical components (CPU, monitor, keyboard)
  • Software: Programs (OS, applications)
  • Liveware: People who use the computer (users)

🖥️ Central Processing Unit (CPU)

ComponentFunction
ALU (Arithmetic Logic Unit)Performs arithmetic (+, -, ×, ÷) and logic operations (AND, OR, NOT)
CU (Control Unit)Controls and coordinates all CPU operations
RegistersSmall, fast memory inside CPU for temporary data

💾 Memory Hierarchy (Fastest → Slowest)

📊 Order of Speed (Top to Bottom = Fast to Slow):
  1. Registers — Inside CPU, fastest, smallest
  2. Cache Memory — Very fast, between CPU and RAM
  3. Main Memory (RAM) — Fast, volatile (loses data when off)
  4. Secondary Storage — Slower, permanent (Hard disk, SSD)
  5. Tertiary Storage — Slowest, for backup (Tape, Cloud)

🔌 Input & Output Devices

📥 Input Devices

  • Keyboard, Mouse, Touch screen
  • Scanner, Webcam, Microphone
  • Joystick, Light pen, OMR/OCR
  • Barcode reader, MICR

📤 Output Devices

  • Monitor (LCD, LED, CRT)
  • Printer (Inkjet, Laser, Dot-matrix)
  • Speaker, Headphone
  • Plotter, Projector
💡 Both Input AND Output Devices

Touch screen, External DVD drive, Modem, Network card

📝 Past Paper Practice

Q1 (2015)

The Central Processing Unit (CPU) of a computer consists of which of the following?

✅ Answer: Arithmetic Logic Unit (ALU), Control Unit (CU), Memory Registers
Q (2024)

Rani wants to store these files in a USB Flash drive: trees.pdf (500 MB), config.txt (534 bytes), nickels.mp4 (2 GB), report.docx (900 KB). Which is the lowest capacity USB sufficient?

✅ Answer: 4 GB

Calculation: Total ≈ 500 MB + 2048 MB + 1 MB + 0.5 MB ≈ 2549 MB ≈ 2.5 GB. So 4 GB drive is the smallest sufficient drive.

  • For storage, convert all to same unit before calculating

Data Representation Methods

Number systems, ASCII, and binary representation

📖 Key Concepts

🔹 Why Binary in Computers?

Computers use binary (0 and 1) because electronic circuits have only two states: ON (1) and OFF (0). This is the most reliable way to store and process data.

🔢 Number Systems

SystemBaseDigits UsedExample
Binary20, 11010₂
Octal80-717₈
Decimal100-915₁₀
Hexadecimal160-9, A-FF₁₆

🔄 Conversion Methods

📌 Decimal → Binary (Division by 2)

Convert 25₁₀ to Binary:

25 ÷ 2 = 12 remainder 1 12 ÷ 2 = 6 remainder 0 6 ÷ 2 = 3 remainder 0 3 ÷ 2 = 1 remainder 1 1 ÷ 2 = 0 remainder 1 Read remainders BOTTOM TO TOP: 11001 Answer: 25₁₀ = 11001₂
📌 Binary → Decimal (Position Value)

Convert 1101₂ to Decimal:

Position: 3 2 1 0 Binary: 1 1 0 1 Value: 8 4 2 1 = (1×8) + (1×4) + (0×2) + (1×1) = 8 + 4 + 0 + 1 = 13₁₀
📌 Binary → Octal (Group 3 bits from right)

Convert 11010101₂ to Octal:

Group from right: 011 010 101 3 2 5 Answer: 11010101₂ = 325₈
📌 Binary → Hexadecimal (Group 4 bits from right)

Convert 11010101₂ to Hex:

Group from right: 1101 0101 D 5 Answer: 11010101₂ = D5₁₆ Hex codes: 10=A, 11=B, 12=C, 13=D, 14=E, 15=F

📝 ASCII Code

🔹 ASCII = American Standard Code for Information Interchange

Used to represent characters using binary numbers.

  • A = 65, B = 66, C = 67... Z = 90
  • a = 97, b = 98, c = 99... z = 122
  • 0 = 48, 1 = 49, 2 = 50... 9 = 57
  • Space = 32
📌 ASCII Example

If 'A' = 65₁₀ = 1000001₂, then:

  • B = 66₁₀ = 1000010₂
  • C = 67₁₀ = 1000011₂
  • D = 68₁₀ = 1000100₂

📝 Past Paper Practice

Q (2018)

Which of the following octal numbers is equivalent to decimal 156?

✅ Answer: 234₈

Working: 156 ÷ 8 = 19 r 4 | 19 ÷ 8 = 2 r 3 | 2 ÷ 8 = 0 r 2 → Read bottom up: 234

Q (2023)

The hexadecimal equivalent of octal 1572₈ is:

✅ Answer: 37A₁₆

Working: First convert octal → binary: 1572₈ = 001 101 111 010₂ = 001101111010₂
Then binary → hex: 0011 0111 1010 = 3 7 A → 37A₁₆

🎯 SHOW YOUR WORKING for partial marks!

  • Even if final answer is wrong, you get marks for method
  • Write each division/remainder clearly
  • Always indicate base subscript: 25₁₀, 11001₂, 19₁₆
  • Verify by converting back to decimal

Logic Gates & Boolean Functions

The building blocks of computer logic

📖 The 5 Main Logic Gates

1️⃣ AND Gate

Rule: Output is 1 ONLY when ALL inputs are 1

Symbol: D-shape | Boolean: A.B or A·B or AB

ABOutput (A AND B)
000
010
100
111

2️⃣ OR Gate

Rule: Output is 1 if AT LEAST ONE input is 1

Symbol: Curved shape | Boolean: A+B

ABOutput (A OR B)
000
011
101
111

3️⃣ NOT Gate (Inverter)

Rule: Inverts the input (0→1, 1→0)

Symbol: Triangle with circle | Boolean: Ā or A'

AOutput (NOT A)
01
10

4️⃣ NAND Gate (NOT AND)

Rule: Opposite of AND. Output is 0 ONLY when ALL inputs are 1

Boolean: (A.B)'

5️⃣ NOR Gate (NOT OR)

Rule: Opposite of OR. Output is 1 ONLY when ALL inputs are 0

Boolean: (A+B)'

🧠 Memory Trick

  • AND = "Multiplication" (1×1=1, anything with 0 = 0)
  • OR = "Addition" (any 1 makes the output 1)
  • NOT = "Opposite/Flip"
  • NAND = "NOT + AND" = flip the AND result
  • NOR = "NOT + OR" = flip the OR result

📝 Past Paper Practice

Q (2020)

Which Boolean expression is equivalent to the output of: (x AND y) OR (NOT y AND x)?

✅ Answer: (x.y) + (ȳ.x)

Trick: Always write inputs first, then operation. AND uses dot (.), OR uses plus (+), NOT uses bar over letter.

Q (2024)

According to the circuit, when A=1, B=1, C=0, what are P and Q outputs?

✅ Always work LEFT TO RIGHT through the circuit

Steps:

  1. Identify each gate type (AND/OR/NOT)
  2. Calculate output of first gate using inputs
  3. Use that output as input for next gate
  4. Continue until you reach the final output

🎯 Marking Scheme for Logic Gates

  • Truth tables — fully complete all 2ⁿ rows (n = number of inputs)
  • For Boolean expressions, use brackets: (A.B) + (Ā.B)
  • Always draw gates with proper shapes (AND=D, OR=curved)
  • Label inputs (A, B, C) and output (X, Y, F)

Operating Systems

The software that manages your computer

📖 Key Concepts

🔹 What is an Operating System (OS)?

An Operating System is system software that manages computer hardware, software resources, and provides services for computer programs. It acts as the bridge between the user and the hardware.

🎯 Main Functions of OS

FunctionDescription
Process ManagementStarts, stops, and schedules programs
Memory ManagementAllocates and frees RAM for programs
File ManagementOrganizes files in folders, handles read/write
Device ManagementControls printers, monitors, keyboards (drivers)
User InterfaceProvides GUI or CLI for user interaction
SecurityManages user accounts, passwords, permissions

💻 Examples of Operating Systems

🖥️ For Computers

  • Windows (10, 11)
  • macOS
  • Ubuntu / Linux
  • Chrome OS

📱 For Mobile Devices

  • Android
  • iOS (iPhone)
  • HarmonyOS
  • Windows Mobile

🔄 Boot Process (What Happens When You Switch ON)

📌 Boot Sequence
  1. BIOS starts (checks hardware - POST)
  2. Boot loader runs (finds OS on disk)
  3. Operating System loads into RAM
  4. Login screen appears
  5. Desktop shows after login
  6. Applications can now be opened

🖱️ Types of User Interfaces

TypeDescriptionExample
GUI (Graphical)Uses icons, windows, mouseWindows, macOS
CLI (Command Line)Type commandsMS-DOS, Linux Terminal
TouchTouch screen interactionAndroid, iOS

Windows, Icons, Menus, Pointer — the four elements of GUI

📝 Past Paper Practice

Q (2024)

The hard drive in a computer is formatted during maintenance. Which of the following should now be first installed in it?

✅ Answer: An Operating System

Why? Without an OS, you can't install any application software. The OS is the foundation needed before anything else can run.

🎯 Marking Tips

  • List ALL OS functions if asked (don't stop at 2-3)
  • For "examples" — give MIX of desktop AND mobile OS
  • Distinguish OS (Windows) from Application (Word)

Electronic Spreadsheet

Numbers, formulas, and data analysis

📖 Key Concepts

🔹 Spreadsheet Software

Software for managing data in rows and columns. Used for calculations, analysis, and charting.

Examples: Microsoft Excel, LibreOffice Calc, Google Sheets

🏗️ Spreadsheet Structure

  • Workbook: The entire file (.xlsx, .ods)
  • Worksheet: A single sheet within the workbook
  • Row: Horizontal (numbered 1, 2, 3...)
  • Column: Vertical (lettered A, B, C... Z, AA, AB...)
  • Cell: Intersection of row and column (A1, B5, etc.)
  • Cell Range: Group of cells (A1:C5)
  • Active Cell: Currently selected cell

🔢 Cell Referencing

ReferenceTypeBehavior When Copied
A1RelativeBoth row & column change
$A1Mixed (column fixed)Only row changes
A$1Mixed (row fixed)Only column changes
$A$1AbsoluteNeither changes (fully locked)
📌 Important Example

If cell C1 contains =A1+B$1 and you copy it to C2:

→ It becomes =A2+B$1

Why? A1 was relative (changed to A2), but B$1 had row locked (stayed as B$1)

Q (2024)

If formula =(B2*G$1/100) + (C2*G$2/100) in D2 is copied to D3, what will D3 contain?

Database

Organizing data efficiently with tables

📖 Key Concepts

🔹 What is a Database?

A database is an organized collection of related data stored electronically. A Database Management System (DBMS) is software used to create and manage databases.

Examples: MS Access, MySQL, Oracle, LibreOffice Base, SQL Server

📊 Database Hierarchy

📌 Structure (Smallest → Largest):

Character → Field → Record → Table → Database

  • Field: A column (e.g., Name, Age)
  • Record: A row (one complete entry)
  • Table: Collection of records
  • Database: Collection of related tables

🔑 Types of Keys

Key TypeDefinitionExample
Primary Key (PK)Uniquely identifies each recordStudentID
Foreign Key (FK)Field that refers to PK of another tableStudentID in Marks table
Composite KeyPK made of 2+ fields combinedStudentID + SubjectID
Candidate KeyAny field(s) that COULD be PKEmail, NIC
💡 Easy Way to Remember

Primary Key = Like your NIC number (unique to you)
Foreign Key = A reference to someone's NIC in another document
Composite Key = When you need TWO things to identify something (like Date + Seat number for cinema ticket)

📋 Data Types

Data TypeUsed ForExample
Text / String / VARCHARNames, addresses, words"Saman Perera"
Number / IntegerCounts, ages, quantities25

🔗 Table Relationships

Three Types:

  • One-to-One (1:1): One record relates to ONE record (rare)
  • One-to-Many (1:N): One record relates to MANY (most common)
  • Many-to-Many (M:N): Many relate to many (needs junction table)

📤 Common Database Operations

📝 Past Paper Practice

Q (2023) - Bus Reservation Database

SEAT_RESERVATION table has: Route_code, Date, Seat_number. What is most suitable as PRIMARY KEY?

✅ Answer: Route_code + Date + Seat_number (COMPOSITE KEY)

Why? Same seat number exists in different buses on different dates. You need ALL THREE to uniquely identify a specific seat reservation.

Q (Common Task)

A new student joins and enrolls in a course. Which tables need updating?

✅ Answer: STUDENT table AND ENROLLMENT table

Steps: First add student details to STUDENT, then add enrollment record to ENROLLMENT (using the new student's ID as FK).

🎯 Marking Tips for Database Questions

  • Always identify keys EXPLICITLY: "Primary key = ID"
  • For inserts, write FULL record format: TableName(field1, field2, ...)
  • Identify ALL affected tables (don't miss any!)
  • For JOIN questions, list ALL tables needed in query

Programming

📖 Key Concepts

🔹 What is a Program?

A set of instructions written in a programming language that tells a computer how to perform a specific task.

🔄 Language Translators

TranslatorFunction

🎯 3 Main Control Structures

Every algorithm uses these three:

  • 1. SEQUENCE: Statements executed in order, one after another
  • 2. SELECTION: Decision-making (IF-THEN-ELSE)
  • 3. ITERATION: Repetition (WHILE, FOR, REPEAT loops)

count = 1 REPEAT DISPLAY count count = count + 1 UNTIL count > 5 Output: 1 2 3 4 5

📝 Past Paper Practice

Q (2020) - Pseudocode Tracing

What is the output?

sum = 0 count = 5 REPEAT sum = sum + count*count count = count - 1 UNTIL count = 0 DISPLAY sum
✅ Answer: 55

Tracing:

  • count=5: sum = 0 + 25 = 25, then count=4
  • count=4: sum = 25 + 16 = 41, then count=3
  • count=3: sum = 41 + 9 = 50, then count=2
  • count=2: sum = 50 + 4 = 54, then count=1
  • count=1: sum = 54 + 1 = 55, then count=0 → STOP
Q (2023) - BMI Algorithm

Write pseudocode to find BMI of 400 students and classify them.

✅ Sample Answer:
BEGIN count = 1 WHILE count <= 400 READ name, grade, weight, height bmi = weight / (height * height) OUTPUT name, grade, bmi IF bmi < 18.5 THEN OUTPUT "Underweight" ELSE IF bmi < 25 THEN OUTPUT "Normal" ELSE IF bmi <= 29.9 THEN OUTPUT "Overweight" ELSE OUTPUT "Obese" ENDIF count = count + 1 ENDWHILE END

  • Pay attention to LOOP CONDITION (=, <, <=, >, >=)
  • Check if counter increments INSIDE or OUTSIDE loop
  • For flowcharts — use CORRECT shapes (diamond for decision!)
  • Show ALL steps in tracing — gets partial marks even if final wrong

System Development Life Cycle

How software systems are built

📖 Key Concepts

🔹 SDLC = System Development Life Cycle

A structured process for designing, developing, testing, and deploying information systems.

📋 The 6 Phases of SDLC (MUST MEMORIZE ORDER!)

#PhaseActivities
1Requirements IdentificationInterviews, observations, questionnaires, prototypes
2Design of SolutionSystem architecture, database design, UI mockups
3Coding (Implementation)Write the actual program code
4TestingUnit → Integration → System → Acceptance
5DeploymentInstall & make operational
6MaintenanceFix bugs, add features, updates

🔄 SDLC Models

📌 Waterfall Model

Linear, sequential approach. Each phase completes before next starts.

  • ✅ Good for: Small, well-defined projects
  • ❌ Limitation: No going back once a phase is complete
  • ❌ User sees system only at the END
📌 Iterative Incremental Model

System built in small portions. Each iteration adds new features.

  • ✅ User sees system EARLY and provides feedback
  • ✅ Easy to add/change requirements
  • ✅ Good for: Large, complex, evolving projects

🧪 Types of Testing (IN ORDER)

#TypeWhat It TestsWho Does It
1Unit TestingIndividual modules/functionsProgrammers
2Integration TestingCombined modules togetherTesters
3System TestingEntire system as a wholeQA Team
4Acceptance TestingDoes it meet user needs?End Users

🚀 Deployment Methods

MethodHow It WorksBest For
DirectSwitch OFF old system, switch ON new immediatelySmall, low-risk systems
ParallelRun BOTH old & new systems simultaneouslyCritical systems (banks)
PhasedDeploy in stages (one module at a time)Large systems
PilotTest with selected group of users firstRisk-averse organizations
⚠️ DON'T CONFUSE!

Direct = Stop old, START new (sudden)
Parallel = BOTH systems running together (safer)
Phased = New system in STAGES
Pilot = Test with a few users FIRST

📝 Past Paper Practice

Q (2024) - Hospital System

A hospital system - requirements are clear, low chance of changes. Which model is suitable?

✅ Answer: Waterfall Model

Why? Requirements are clear and stable, so the linear approach works well. No need for iterations.

Q (Common Pattern)

An LMS - want to add features based on user feedback. Which model?

✅ Answer: Iterative Incremental Model

Why? Continuous feedback & adding features = perfect for iterative approach.

🎯 SDLC Marking Tips

  • ALWAYS list phases in ORDER (don't shuffle)
  • For "who does what" — match person to phase
  • Justify deployment choice with REASON
  • Distinguish testing types clearly

Use of Multimedia

Images, audio, video, and animation

📖 Key Concepts

🔹 What is Multimedia?

The use of multiple forms of media together: text, images, audio, video, and animation.

🖼️ Image Concepts

🔹 Pixel

A pixel is the smallest dot of color in a digital image. Images are made of millions of pixels.

🔹 Resolution

Number of pixels in an image (e.g., 1920 × 1080 = 2,073,600 pixels)

Higher resolution = better quality = larger file size

🔹 Color Depth (Bits Per Pixel - bpp)

Number of bits used to store color for each pixel.

  • 1 bpp = 2 colors (B&W)
  • 4 bpp = 16 colors
  • 8 bpp = 256 colors
  • 16 bpp = 65,536 colors
  • 24 bpp = 16.7 million colors (True Color)

Formula: Colors = 2^bpp

Marking Scheme & Time Management

Score maximum marks with smart strategies

⏰ Time Management Strategy

📝 Paper I (40 MCQs - 1 hour)

📌 Recommended Time Distribution
  • Total time: 60 minutes
  • Per question: 1.5 minutes average
  • Read time: 5 minutes (initial scan)
  • Easy questions first: 30 minutes (do all easy ones)
  • Hard questions: 20 minutes (tackle difficult ones)
  • Review/Mark: 5 minutes (transfer to answer sheet)

📋 Paper II (Q1 + 4 from Q2-Q7 = 60 marks)

📌 Recommended Time Distribution
  • Total time: 2-3 hours
  • Read all questions: 10 minutes
  • Choose 4 best questions: 5 minutes
  • Q1 (compulsory, 20 marks): 40 minutes
  • Q2-Q7 (4 × 10 marks): 25 minutes each = 100 minutes
  • Review & corrections: 15 minutes
💡 Smart Strategy

For Paper II:

  1. Quickly read ALL 7 questions first
  2. Mark which 4 you can answer BEST
  3. Do Q1 first (compulsory)
  4. Then do your STRONGEST chosen question
  5. End with your WEAKEST chosen question
  6. This way you score maximum on easy ones first!

💯 Marking Scheme Secrets

🎯 How Marks Are Awarded:

  • 1 mark per CORRECT TERM (technical vocabulary matters!)
  • 1 mark per VALID POINT (each idea = 1 mark)
  • 1 mark per LABELED DIAGRAM ELEMENT
  • 1 mark per CORRECT FORMULA STEP (show working!)
  • 1-2 marks for EXAMPLES (always give specific examples)

✅ Tips to MAXIMIZE Marks

For All Questions:

  • 📌 ALWAYS write the question number and part (e.g., 1(a), 2(b))
  • 📌 Use TECHNICAL terms ("CPU" not "the brain part")
  • 📌 Give SPECIFIC examples ("Microsoft Word" not "a word program")
  • 📌 SHOW your working for calculations
  • 📌 DRAW clear, labeled diagrams
  • 📌 Use COLORED pens for diagrams (if allowed)
  • 📌 Write neatly — examiners can't mark what they can't read
